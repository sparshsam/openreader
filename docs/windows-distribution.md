# Windows Distribution Strategy

**Version:** 1.2.0+  
**Status:** MSIX/App Installer migration — Store reserved

## Overview

OpenReader has migrated from Inno Setup self-updating to **MSIX** — Microsoft's
modern packaging format — with multiple distribution channels. The app no longer
downloads or runs installers from within itself; Windows handles updates.

## ⚠️ Frozen Identity (MSIX / Microsoft Store)

The following values are **permanently frozen** and must never change after
the first public release. Changing **Identity Name** or **Publisher** will break
upgrade continuity for all existing users.

| Field | Frozen Value |
|-------|-------------|
| Store Display Name | `OpenReader` |
| Identity Name | `SparshSam.OpenReader` |
| Publisher | `CN=E6186421-BF8A-47E0-A89C-0F513DFF91C0` |
| Publisher Display Name | `Sparsh Sam` |
| Package Family Name | `SparshSam.OpenReader_yh0byntbzd2qw` |
| Store ID | `9MXDVW2645LL` |
| Executable | `OpenReader.exe` |

> **Identity Name** and **Publisher** — never change after first release.
> **Package Family Name** — auto-generated by Windows from Identity + Publisher.
> **Store Display Name** — may be updated in future without breaking upgrades.
> Never use `com.sparshsam.openreader`, `com.sparshsam.pdfreader`, or placeholder
> Publisher values.

## Distribution Channels

### Primary: Microsoft Store

**Store ID:** `9MXDVW2645LL`

OpenReader is reserved in the Microsoft Store. Users can search for "OpenReader"
or install directly via:

```
ms-windows-store://pdp/?productid=9MXDVW2645LL
```

**Store advantages:**
- Built-in trust (Store-signed)
- Automatic updates via Windows Update
- Discovery through Store search
- Clean install/uninstall
- No Developer Mode required

**Store submission workflow:**
1. Build unsigned MSIX (`OpenReader.msix`)
2. Submit via Partner Center
3. Microsoft signs and distributes
4. Users receive updates through Windows Update

### Secondary: GitHub Releases (MSIX + Setup.exe)

| Asset | Format | Status |
|-------|--------|--------|
| `OpenReader.msix` | MSIX | ✅ Primary — recommended distribution |
| `OpenReader-Setup.exe` | Inno Setup | ⚠️ Legacy/manual recovery only — no updates |
| `OpenReader-Windows.zip` | Portable ZIP | ⚠️ Legacy/manual recovery only |

**MSIX from GitHub Releases:** Unsigned — requires Windows Developer Mode for
sideloading until a code-signing certificate is procured or Store submission
is active.

**Legacy Setup.exe:** Retained for manual recovery. No longer supports in-app
update triggering. Requires admin rights (UAC).

## App Installer (Future — Not Yet Active)

Automatic updates via Windows App Installer are **not yet implemented**.
The `packaging/msix/AppInstaller.xml` template exists as infrastructure
preparation.

**Permanent update URI (must never change after public release):**
```
https://downloads.openreader.app/stable/OpenReader.appinstaller
```

**Hosting requirements:**
- HTTPS with valid TLS certificate
- Content-Type `application/xml` for `.appinstaller`
- Content-Type `application/octet-stream` or `application/x-msix` for `.msix`
- File serving: `OpenReader.appinstaller` + `OpenReader.msix`

**Update flow (when implemented):**
1. Windows App Installer checks the URI on launch
2. Background periodic checks (managed by Windows)
3. Download + install silently
4. Next launch uses new version

## Code Signing

### Microsoft Store Signing (Preferred)
Store-signed packages are trusted on all Windows systems automatically.

### Third-Party Certificate (Alternative)
An OV or EV code-signing certificate enables sideloading without Developer Mode:
- OV: ~$200–300/year
- EV: ~$300–500/year (immediate SmartScreen reputation)

### Unsigned Behavior
- Requires Developer Mode (Settings → Privacy & security → For developers)
- SmartScreen warnings for unsigned executables
- MSIX: `Add-AppxPackage -Path .\OpenReader.msix` with Developer Mode only

## Legacy Inno Setup Installer

| Aspect | MSIX | Inno Setup |
|--------|------|------------|
| **Admin required** | No (per-user) | Yes |
| **Update mechanism** | Store / App Installer | Manual download only |
| **Clean uninstall** | Automatic | Partial |
| **File association** | Declared in manifest | Registry-based |
| **Self-update from app** | No (Windows handles it) | No (removed in v1.2.0) |

## Migration Path

1. **v1.1.x users:** Install OpenReader MSIX alongside existing installation.
   User settings are in `%APPDATA%` (shared across versions).
2. **New users:** Prefer Microsoft Store when available; otherwise MSIX from
   GitHub Releases (enable Developer Mode if unsigned).
3. **Future:** Store exclusively once Store submission is active.
