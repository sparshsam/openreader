# PDFReader by Sparsh v1.0.6 — Reliability & Polish

**Release:** [v1.0.6](https://github.com/sparshsam/pdfreader-by-sparsh/releases/tag/v1.0.6)
**Published:** June 2026
**Platforms:** Windows (installer + portable), macOS (Apple Silicon + Intel)

---

A production-quality maintenance release focused on ironing out the last rough edges in the Windows installation and open flow.

### What's New

**🔧 File → Open Reliability — Fixed**
- Resolved a `QFileDialog` issue on frozen Windows builds where the native file dialog would fail silently. A 3-tier fallback chain now ensures you can always open a PDF.
- **Single-instance IPC routing** — double-clicking a PDF file while the app is running now opens it in a new tab via `QLocalServer` instead of launching a second window.

**🖼 Application Icon — Fixed**
- The app icon now appears correctly everywhere: taskbar, title bar, Alt+Tab switcher, and Start Menu. A multi-tier fallback path ensures the icon is found regardless of PyInstaller layout.

### Why v1.0.6 Matters

This release closes a chapter. The original v1.0.5 fix for the open flow landed, but the frozen builds were built without the `assets/` directory, so the icon was missing and the dialog fallback wasn't exercised. v1.0.6 verifies everything end-to-end:

- The release workflow packages `assets/` into frozen builds
- The icon fallback chain covers every known PyInstaller layout
- The open dialog fallback chain ensures PDFs open in every scenario
