# v1.1.0 — Premium Visual Polish & Desktop Maturity

> **Status:** Planning phase — no implementation started.
> **Target:** Next minor release after v1.0.6 (current stable).
> **Base:** `main` at `6c82278` (v1.0.6).

---

## Scope

### Core — Premium Visual Polish

- [ ] **Better tab styling** — refine tab appearance with tighter spacing, hover states, close-button visibility, and drag reordering polish.
- [ ] **Recent files / start screen** — replace the blank startup with a recent-files landing page showing thumbnails of recently opened documents.
- [ ] **Thumbnail / sidebar exploration** — add a collapsible sidebar with page thumbnails for visual document navigation.
- [ ] **Updated UI screenshots** — after UI polish is applied, update README with fresh screenshots showing the polished app (empty state, open PDF, multi-tab, About dialog).

### Rendering Quality

- [ ] **Rendering quality refinements** — improve page rendering fidelity (anti-aliasing, HiDPI scaling, smooth zoom transitions).

### Distribution & Packaging

- [ ] **Signed Windows binaries research** — investigate options for code-signing the Windows installer (EV certificate, Azure Key Vault, GitHub's `sigstore` / `dlc`).
- [ ] **macOS DMG packaging follow-up** — refine the macOS distribution flow: create a `.dmg` installer instead of raw `.zip`, add background art, volume icon, and drag-to-Applications shortcut.

### v1.0.6 Freeze

- **v1.0.6** is the current stable release and will receive only critical bug fixes.
- Non-critical improvements, visual polish, and new features are deferred to v1.1.0+.

---

## Excluded from v1.1.0

The following items remain on the longer-term radar but are **out of scope** for this milestone:

- Local AI summarization (Ollama / llama.cpp)
- Code signing (requires research — tracked under v1.1.0 scope above)
- Cross-platform Linux support
- Plugin system
- PDF timeline / version history
- Cryptographic document anchoring (Base blockchain)
- Stronger sandboxing guidance

---

## How to Contribute / Branch Strategy

- All new work for v1.1.0 branches off `main`.
- Branch naming: `feat/<short-description>` or `polish/<short-description>`.
- Keep PRs small and focused on a single item from the scope list.
- Do not start implementation before the v1.1.0 planning is finalized and approved.
